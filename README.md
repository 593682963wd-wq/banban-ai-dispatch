# 带班AI分飞机

放行签派席位智能均衡分配系统。代班主任上传次日动态列表，AI 自动把每架飞机当日的全部航班作为一个整体，均衡分配到 **放行签派一 / 放行签派二** 两个席位。

本地端（PySide6 桌面）与网页端（Streamlit）共用同一份 `core/` 纯算法层，方便交付公司开发同事二次集成。

---

## 一、分配规则

- **分配单位 = 飞机（机号）**：一架飞机当天所有航班绑定到同一席位，不拆分单个航班。
- **多目标加权最优**，同时兼顾以下维度的两席位均衡：

| 维度 | 说明 |
|---|---|
| 总任务量 | 两席位航班总数尽量接近 |
| 时段忙闲 | 按分时切分点（13:30 / 16:30）划分上午、下午，两席位都不过载 |
| C 类机场 | 高高原/复杂机场航班均分（最受关注） |
| B 类机场 | 次复杂机场航班均分 |
| C/B 同目的地 | 同一目的地机场（尤其 C 类、B 类，如丽江）尽量拆到两个席位 |
| 长沙出港 | 长沙始发航班两席位尽量接近 |
| 讲解量 | 长沙出港、C 类目的地、B 类目的地命中任一项即计 1 次讲解量，尽量均衡 |
| 空任务飞机 | 固定 17 架机队中当日无航班任务的飞机也均分到两个席位 |
| 区域分布 | 东北/华东/云南/西南… 各方向不集中在一个席位 |
| 过夜地 | 长沙/昆明/无锡等过夜目的地均衡；同一过夜地末班落地前后 2 小时内的飞机尽量拆到两个席位 |
| 相近时刻冲突 | 同一时刻附近起飞的飞机尽量分到不同席位 |
| 交接班窗口 | 交接班时段两席位任务都不扎堆 |

> 历史人工分配通常牺牲总数均衡（差 0~4）去换取 C/B 类与区域均衡；本系统在所有维度上同时优化，实测综合均衡度优于人工（0605 综合得分 119 vs 人工 201，越低越均衡）。

切分点默认规则：6/1 起 13:30，之前 16:30；界面可手动指定。

固定机队默认清单在 `core/fleet.py`，当前 17 架：

`305L、306C、300Z、302Y、303M、30A2、30AM、30AN、30EH、321U、322C、325Q、32Q6、8432、8983、8285、8318`

后续新增飞机时，补充 `KNOWN_FLEET_TAILS` 即可。

## 二、目录结构

```
banban_ai_dispatch/
├── core/                  # 纯算法层（无 UI 依赖，可独立交付）
│   ├── airports.py        # 机场分类（C/B 类、区域、过夜地、ICAO→中文名）
│   ├── fleet.py           # 固定机队清单，用于补齐空任务飞机
│   ├── models.py          # Flight / Aircraft / AllocationConfig / SeatPlan / AllocationResult
│   ├── parser.py          # 动态列表 Excel 解析（列名容错、按机号聚合）
│   ├── allocator.py       # 多目标分配引擎（全枚举 + 局部搜索兜底）
│   └── excel_writer.py    # 分配结果导出 Excel（席位灰/蓝配色）
├── gui/
│   └── main_window.py     # PySide6 桌面主窗口
├── main.py                # 桌面入口
├── app.py                 # 网页端（Streamlit，与网页工具台 tools_dispatch 同源）
├── requirements.txt
└── README.md
```

## 三、运行

```bash
# 桌面端（含 PySide6）
pip3 install -r requirements-desktop.txt
python3 main.py

# 网页端（仅 pandas/openpyxl/streamlit，无 Qt）
pip3 install -r requirements.txt
python3 -m streamlit run app.py
```

## 四、算法说明

`allocator.allocate(aircrafts, config)`：

1. 解析动态列表时先补齐 `core/fleet.py` 中的固定 17 架机队，缺席机号按空任务飞机处理。
2. 把有航班的飞机分为 active，无航班（停场/备用）记入 `idle_tails`，并轮流分配到两个席位。
3. active ≤ 20 架时**全枚举** `2^(N-1)` 种二分方案（固定首架去镜像对称），取综合得分最低者。
4. 超过阈值时退化为多次随机重启的**局部搜索**。
5. 每种方案按上表各维度的两席位差值加权求和，得到综合得分（越低越均衡）。同一过夜地若末班落地时刻相距不超过 120 分钟，分到同一席位会额外计入“过夜落地窗口冲突”。
6. 输出时让 **放行席位1 = 较大的一组**，与公司系统灰=席位1、蓝=席位2 的约定一致；空任务飞机按固定机队顺序均分补入两个席位。

`evaluate_split(...)` 可对任意给定分配（如历史人工分配）计算同口径指标，便于对照验证。

## 五、输入格式

动态列表 Excel 首行表头，需包含（列名支持别名容错）：

| 标准字段 | 可识别表头示例 |
|---|---|
| 机号 | 机号 / 机尾号 / 注册号 |
| 机型 | 机型 / 机型代码 / 型别 |
| 始发 | 始发 / 始发站 / 起飞机场 |
| 局飞 | 局飞 / 计划起飞 / 预计起飞 |
| 到达 | 到达 / 到达站 / 目的地 |
| 局达 | 局达 / 计划到达 / 预计到达 |

机场均为 ICAO 四字码。
